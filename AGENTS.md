# Інструкції для агентів

- Після будь-яких змін у репозиторії обов'язково запускайте `pytest` або інші релевантні тести, щоб переконатися, що все працює коректно.
- Якщо ви змінюєте функціональність, яку вже покривають тести, або додаєте новий критичний функціонал, оновіть наявні тести чи додайте нові, щоб вони відображали зміни.
- Це застосунок для кінцевих користувачів Windows, які запускають `.exe`, зібраний у GitHub Actions, тому уважно стежте за конфігурацією збірки та переліком пакетів, що потрапляють у білд.
- Після внесення змін перевіряйте, чи впливають вони на процес збірки, і за потреби оновлюйте налаштування GitHub Actions.
- Якщо додаєте нові допоміжні виконувані файли або змінюєте їхні назви/розташування, переконайтеся, що workflow GitHub Actions збирає та пакує їх разом з основним `.exe`, і задокументуйте такі вимоги у відповідних інструкціях.
